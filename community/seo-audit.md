# SEO + discoverability audit (IRO-277)

Internal Growth/DevRel operating doc. Audit of on-site SEO for the MkDocs site
(`ironsecco.github.io/ironclaw`) and the GitHub repo. Fixes for the concrete gaps
landed in the same PR as this file. Not published to the docs site.

Date: 2026-07-02. Auditor: Growth (WS-E). Build verified with `mkdocs build --strict`.

## Why this matters (and why it is not launch-gated)

Organic discovery compounds and is independent of the IRO-40 launch fire-gate.
Someone searching "self-hosted AI agent sandbox", "gVisor agent isolation", or
"BYO-model agent runner" should find IronClaw whether or not we have posted the
launch announcement. On-site SEO is safe to ship now; it makes no public claim we
have not already shipped and creates no external account.

### Target keyword set (from the comparison page competitors, IRO-151)

Primary: `self-hosted AI agent`, `gVisor sandbox / agent isolation`,
`sandboxed AI agents`, `human approval gateway`, `zero-credential agent`,
`BYO-model / local model agent runner`, `AI agent security`,
`AGPL open-source agent platform`. These now appear across page titles and meta
descriptions, not just the homepage.

## Fixes landed in this PR

1. Added `title:` + `description:` front-matter to 14 pages that lacked it:
   benchmarks, building, contract, deployment, ironclaw-explained, observability,
   pr-review-process, release-runbook, roadmap, social-proof, troubleshooting,
   vault, writing-a-channel-adapter, reference/api. Descriptions are unique,
   keyword-bearing, and free of em/en dashes (IRO-254). Note: keep a `: ` (colon +
   space) out of a plain-scalar description or MkDocs silently drops it back to the
   site-wide default; reword or quote.
2. Added `vault.md` to the nav (Security & Trust) to end its orphan state.
3. Verified: strict build passes, canonical + per-page `description` + `og:description`
   render correctly, sitemap lists 30 URLs.

Already in place before this audit (no change needed): sitemap, robots.txt,
canonical, OG/Twitter cards, homepage JSON-LD. The site was in good shape; the
gap was per-page meta descriptions and one orphaned page.
